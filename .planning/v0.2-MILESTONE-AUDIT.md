## Integration Findings

### Finding INT-1 (HIGH): WeatherDaemon `load_config` not at module level

**File:** [weather_daemon.py](src/cta_eta/data_collection/orchestration/weather_daemon.py)
**Lines:** 39 (imports), 124 (`__init__` call), 825 (`if __name__ == "__main__"` local import)

`load_config` is imported only inside the `if __name__ == "__main__"` block (line 825). Inside `__init__` at line 124, `config = load_config()` is called when no config is passed — but `load_config` is not in module-level scope.

**When it fails:** Any caller that `import`s `WeatherDaemon` from another module and instantiates `WeatherDaemon(logger)` without `config=` will get `NameError: name 'load_config' is not defined`.

**Why the systemd path currently works:** `python -m` sets `__name__ == "__main__"`, so the local import at line 825 runs, adding `load_config` to module globals *before* `WeatherDaemon(logger)` is called.

**Contrast:** `train_position_daemon.py` line 42 correctly imports `load_config` at module level.

**Risk:** If `WeatherDaemon` is ever imported programmatically (e.g., in a test without mocking `create_journal_writer`, or in a future integration script), it will fail. This is the same class that was just refactored in Phase 10.

**Fix:** Add `load_config` to the module-level import at line 39.

### Finding INT-2 (MEDIUM): `compact.py` uses `datetime` where `date` needed — malformed cloud paths

**File:** [compact.py](src/cta_eta/data_collection/compaction/compact.py)
**Line:** 574

```python
target_date = reprocess_date or (datetime.now(tz=UTC) - timedelta(days=1))
```

`datetime.now(tz=UTC) - timedelta(days=1)` produces a **`datetime`** object. Its `.isoformat()` returns `"2026-02-25T03:00:00.848342+00:00"` instead of `"2026-02-25"`.

**Downstream breakage:**
| Location | What's Written | Should Be |
|----------|---------------|-----------|
| Line 306: local staging dir | `date=2026-02-25T03:00:00+00:00/` | `date=2026-02-25/` |
| Line 341: cloud URL | `…/train_positions/date=2026-02-25T03:00:00+00:00/data.parquet` | `…/train_positions/date=2026-02-25/data.parquet` |
| Line 400: sidecar filename | `compaction-2026-02-25T03:00:00+00:00-train_positions.json` | `compaction-2026-02-25-train_positions.json` |

Cloud objects stored with non-standard hive partition paths break downstream model training (hive readers expect `date=YYYY-MM-DD`). Local paths are consistent with each other so `cta-monitor` still works (it reads the same malformed `date_str` from the sidecar JSON).

**Reprocess path is unaffected:** `reprocess_date = date.fromisoformat(args.reprocess)` is always a `date` object.

**Fix:** [compact.py:574](src/cta_eta/data_collection/compaction/compact.py#L574) — add `.date()`:
```python
target_date = reprocess_date or (datetime.now(tz=UTC) - timedelta(days=1)).date()
```

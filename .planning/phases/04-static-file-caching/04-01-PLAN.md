Create new API module following established patterns from api_cta_track_shape.py:

**API Endpoint:**
- Base: "https://data.cityofchicago.org/api/v3/views/{dataset_id}/query.json"
- Dataset ID: "3tzw-cg4m" (from api_stations_weather.py example)
- Authentication: X-App-Token and X-App-Secret headers from config secrets

**Functions to implement:**

1. `_get_chidata_headers(cfg)` - Extract Chicago Data credentials from config, raise ValueError if missing (reuse pattern from api_cta_track_shape.py)

2. `get_cta_stations(client: httpx.Client) -> list[dict[str, Any]]` - Fetch stations from API
   - Decorated with @stamina.retry(on=httpx.HTTPStatusError, attempts=MAX_RETRY_ATTEMPTS)
   - Decorated with @log_api_call(logger)
   - GET request to query.json endpoint with authentication headers
   - Returns raw JSON response

3. `normalize_cta_stations(raw_data: list[dict]) -> list[dict[str, Any]]` - Normalize API response
   - Extract fields: station_id (as "id"), longname (as "name"), address, lines, the_geom coordinates
   - Convert GeoJSON point to flat latitude/longitude fields (coordinates[1] = lat, coordinates[0] = lon)
   - Return list of normalized dicts matching format from api_stations_weather.py example:
     ```python
     {
         "id": "900",
         "name": "Howard",
         "address": "1649 W. Howard Street",
         "lines": "Red, Yellow, Purple, Evanston Express",
         "latitude": 42.019063,
         "longitude": -87.672892
     }
     ```

4. `get_stations_cache() -> CachedData[list[dict[str, Any]]]` - Factory function for cache
   - Load config with load_config()
   - Extract cache directory and stations_ttl from config[cache]
   - Create cache file path: Path(cache_dir) / "cta_stations.json"
   - Define fetch function that creates httpx.Client, calls get_cta_stations + normalize_cta_stations
   - Return CachedData instance with cache_file, ttl, and fetch_fn

**Constants:**
- CHICAGO_DATA_VIEWS_BASE: Final[str] = "https://data.cityofchicago.org/api/v3/views"
- CTA_STATIONS_DATASET_ID: Final[str] = "3tzw-cg4m"
- MAX_RETRY_ATTEMPTS from config.retry.max_retry_attempts (default 10)

**Imports:**
- from __future__ import annotations (first line after docstring)
- Standard library: from typing import Any, Final; from pathlib import Path
- Third-party: import httpx; import stamina
- Own modules (### OWN MODULES comment): config, logging, cache

**Error handling:**
- Raise ValueError with descriptive message if CHIDATA_APP_TOK or CHIDATA_APP_SECRET missing
- Let stamina decorator handle HTTPStatusError retries
- Propagate other exceptions (JSON decode errors, network failures)

**What to avoid and WHY:**
- Do NOT use api_stations_weather.py's /resource/ endpoint URL - it returns different JSON structure. Use /api/v3/views/{id}/query.json for consistent schema (WHY: query.json provides structured response with predictable field names)
- Do NOT skip normalization - API returns nested GeoJSON that needs flattening (WHY: Downstream code expects flat dict with latitude/longitude fields)
- Do NOT create httpx.Client in module scope - use dependency injection pattern (WHY: Caller manages client lifecycle for connection pooling and testing)

  <behavior>
Detect missed collection windows and generate metadata for Parquet datasets:

**Gap Definition:**
A gap occurs when the time between polls exceeds the configured poll_interval by a threshold (e.g., 2x interval to account for retry attempts and network latency).

**Test Cases (input → expected output):**

1. **No gap** (within threshold):
   - last_poll: 1000.0, current: 1015.0, interval: 15 → is_gap=False

2. **Retry exhaustion gap** (exceeded threshold):
   - last_poll: 1000.0, current: 1400.0, interval: 15 → is_gap=True, duration=400s, reason="retry_exhausted"

3. **Downtime gap** (daemon was stopped):
   - last_poll: 1000.0, current: 10000.0, interval: 15 → is_gap=True, duration=9000s, reason="downtime"

4. **Multiple interval gap** (long network outage):
   - last_poll: 1000.0, current: 1200.0, interval: 15 → is_gap=True, duration=200s, missed_polls=~13

5. **Edge case - exactly at threshold**:
   - last_poll: 1000.0, current: 1030.0, interval: 15, threshold: 2.0 → is_gap=False (30s = 2×15s exactly)

6. **Edge case - first poll ever**:
   - last_poll: None, current: 1000.0 → is_gap=False (no previous poll to compare)

**Output Schema:**
```python
{
    "is_gap": bool,
    "gap_start_timestamp": float | None,  # Same as last_poll_timestamp if gap
    "gap_end_timestamp": float | None,    # Same as current_timestamp if gap
    "gap_duration_seconds": float | None,
    "gap_reason": str | None,  # "retry_exhausted", "downtime", "unknown"
    "missed_poll_cycles": int | None,  # Estimated missed polls
}
```
  </behavior>
  <implementation>
After tests pass, implement:

1. **create detect_gap() function** in new gap_detection.py module:
   - Calculate time delta between polls
   - Compare against threshold (configurable, default 2× interval)
   - Determine gap reason heuristically:
     * delta < 10 minutes: "retry_exhausted" (retries took too long)
     * delta >= 10 minutes: "downtime" (daemon was stopped)
   - Calculate missed_poll_cycles: max(0, int(delta // interval)). Number of poll cycles that would have been scheduled in (last_poll, current]; do not use "floor(...) - 1".
   - Return gap metadata dict

2. **Integrate into TrainPositionDaemon**:
   - At start of each collection cycle, call detect_gap(last_poll_timestamp, time.time(), poll_interval). If last_poll_timestamp is 0 or None (first run), no gap.
   - If gap detected, log warning with gap metadata and set pending_gap_metadata. On the next successful append_batch, pass pending_gap_metadata to ParquetWriter and clear it. Do not write a separate Parquet batch just for the gap; attach to the next successful poll's write. Same pattern for retry-exhaustion (Plan 07-01 sets pending_gap_metadata when TRANSIENT retry is exhausted) and for restart (Plan 07-03).

3. **Extend ParquetWriter with metadata support**:
   - Add optional metadata parameter to append_batch()
   - Store gap metadata in Parquet file custom metadata
   - Metadata attached at file level (not row level) for efficiency
  </implementation>

  <name>Task 1: Enhance shutdown state preservation in AsyncBaseDaemon</name>
  <files>src/cta_eta/data_collection/orchestration/daemon_async.py</files>
  <action>
Enhance AsyncBaseDaemon's shutdown sequence to guarantee state persistence before exit:

**Current behavior:** Signal handlers (SIGTERM/SIGINT) set self.running = False and notify _shutdown event. The run loop exits, then stop() is called which calls _save_state().

**Enhancement needed:**
1. **Flush pending writes before state save**: In _pre_shutdown_hook(), if the daemon has a storage object with a flush() method, call it (e.g. getattr(getattr(self, "storage", None), "flush", lambda: None)()). Current ParquetWriter has no flush (writes are immediate); this is for future buffered writers. No-op if flush does not exist.
2. **Add shutdown timeout**: If stop() takes >30 seconds (configurable), log warning but still complete state save
3. **Verify state save success**: Log confirmation after _save_state() completes, log error if save fails
4. **Handle shutdown during retry**: If shutdown signal received during retry/backoff (including 07-01's DAILY_QUOTA probe intervals and midnight sleep), the run loop and any long sleeps must use self.sleep or chunked asyncio.sleep with _shutdown/self.running checks so shutdown completes in reasonable time. 07-01 is responsible for making 102 sleeps interruptible; 07-03's "Handle shutdown during retry" encompasses those periods.

**Implementation approach:**
- Add _pre_shutdown_hook() as an overridable method (default no-op in base) for daemon-specific cleanup including optional storage.flush()
- Call _pre_shutdown_hook() in stop() before _save_state()
- Wrap _save_state() in try/except to log errors but not crash shutdown
- Add shutdown_timeout_seconds config parameter (default: 30)
- **Apply loaded state on init:** In AsyncBaseDaemon.__init__, replace _load_state() with: state = self._load_state(); self._apply_state(state if state is not None else {}). Add _apply_state(self, state: dict) -> None as an overridable no-op in the base. TrainPositionDaemon overrides _apply_state to set self.last_poll_timestamp, self.total_records_collected, self.current_poll_count from state (defaults 0 when missing). TrainPositionDaemon.__init__ must not overwrite those after super()—remove or drop the explicit self.last_poll_timestamp = 0.0 etc. so _apply_state is the single source.

**What to avoid and WHY:**
- Don't make _save_state() blocking forever - use timeout to prevent hung shutdowns
- Don't skip state save on exceptions - state preservation is critical; log error but complete the save
- Don't assume storage has flush - use getattr to avoid AttributeError; current ParquetWriter has no flush

<task type="auto">
  <name>Task 2: Implement restart gap detection and reporting</name>
  <files>src/cta_eta/data_collection/orchestration/train_position_daemon.py</files>
  <action>
Enhance TrainPositionDaemon to detect and report downtime gaps on restart:

**Restart sequence:**
0. **Apply loaded state to daemon instance:** AsyncBaseDaemon._load_state() reads and returns a state dict; the return was previously ignored. In __init__, after state = self._load_state(), if state is not None, call self._apply_state(state). Add _apply_state(self, state: dict) as an overridable method (base: no-op). TrainPositionDaemon overrides _apply_state to set self.last_poll_timestamp, self.total_records_collected, self.current_poll_count from state. This runs before _check_restart_gap() so last_poll_timestamp is correct.
1. **Calculate downtime gap** using gap_detection.detect_gap(last_poll_timestamp, time.time(), poll_interval). last_poll_timestamp comes from _apply_state (or 0 on first run).
2. **Log downtime gap** if detected:
   - Log warning with gap duration, gap_start, gap_end
   - Include "reason: downtime" to distinguish from retry gaps
3. **Flag gap in first collection** after restart: Pass gap metadata to ParquetWriter on first successful poll; store as file-level metadata. Clear after first poll. (Same pending_gap_metadata pattern as Plan 07-02 for retry-exhaustion.)
4. **Resume normal polling** after gap reported

**Implementation approach:**
- TrainPositionDaemon overrides _apply_state (from Task 1) to set self.last_poll_timestamp, self.total_records_collected, self.current_poll_count from state. base __init__ calls _apply_state after _load_state before any subclass logic that needs state.
- Add _check_restart_gap() called at end of __init__ (after _apply_state has run)
- Import detect_gap from gap_detection (Plan 07-02)
- Store restart_gap_metadata if gap detected; pass to storage on first successful poll; clear after first poll (same pending_gap_metadata pattern as 07-02)

**What to avoid and WHY:**
- Don't report restart gap on every poll - flag it once on first successful collection after restart, then clear
- Don't block daemon start on gap calculation - calculate gap, log it, but proceed to polling immediately
- Don't treat restart gaps as errors requiring retry - they're expected after controlled shutdown, just log and flag for ML
- Don't assume saved state exists - handle case where daemon starts for first time (no previous state)
  </action>

4. Mock long downtime (24 hours) → verify gap metadata includes correct duration
Tests should verify gap logged and gap metadata passed to storage exactly once.

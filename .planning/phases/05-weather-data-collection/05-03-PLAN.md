2. **Update _collect_weather_cycle() to use aiometer.run_on_each()**:
   Replace manual for-loop over grid points with rate-limited batch:
   ```python
   # From RESEARCH.md Pattern 2
   async def _fetch_with_fallback(grid_point):
       nws_grid, om_grid, lat, lon = grid_point
       nws_data, om_data = await self._fetch_weather_for_grid_point(client, nws_grid, om_grid, lat, lon)

       # Fallback to OpenWeatherMap if either source failed
       owm_data = None
       if isinstance(nws_data, Exception) or isinstance(om_data, Exception):
           owm_data = await get_openweathermap_current(client, lat, lon)

       return (nws_data, om_data, owm_data, lat, lon, timestamp)

   unique_points = await self._get_unique_grid_points()

   # Rate limit: 10k/day = 6.9/min, use 6/min for margin
   results = await aiometer.run_on_each(
       _fetch_with_fallback,
       unique_points,
       max_per_second=0.1,  # 6 per minute = 0.1 per second
       max_at_once=3        # Max 3 concurrent requests
   )
   ```

3. **Handle OpenWeatherMap fallback logic**:
   - Only call OpenWeatherMap if NWS OR Open-Meteo failed (isinstance check for Exception)
   - Log: "Falling back to OpenWeatherMap for grid point {lat},{lon} due to {source} failure"
   - If all three sources fail, log error and skip that grid point

4. **Update imports**:
   - Add: `from cta_eta.data_collection.apis.api_weather_openweathermap import get_openweathermap_current`
   - Add: `import aiometer`

**What to avoid and WHY:**

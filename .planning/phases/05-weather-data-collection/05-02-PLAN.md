  <behavior>
Function: merge_weather_sources(nws_data: dict | None, om_data: dict | None, owm_data: dict | None = None) -> dict

Expected behavior:
1. **Both sources present**: Merge NWS + Open-Meteo preserving all variables
   - Input: nws_data={'timestamp': '2026-01-19T12:00', 'temperature_f': 35.0, 'wind_speed_mph': 10.0}, om_data={'timestamp': '2026-01-19T12:00', 'visibility_m': 10000.0, 'wind_speed_mph': 11.0}
   - Output: {'timestamp': '2026-01-19T12:00', 'temperature_f': 35.0, 'wind_speed_mph': 10.0, 'visibility_m': 10000.0} (NWS wind_speed preferred)

2. **NWS missing**: Use Open-Meteo + fallback to OpenWeatherMap
   - Input: nws_data=None, om_data={'timestamp': '2026-01-19T12:00', 'temperature_f': 34.0}, owm_data={'timestamp': '2026-01-19T12:00', 'humidity_percent': 75.0}
   - Output: {'timestamp': '2026-01-19T12:00', 'temperature_f': 34.0, 'humidity_percent': 75.0}

3. **Open-Meteo missing**: Use NWS + fallback to OpenWeatherMap
   - Input: nws_data={'timestamp': '2026-01-19T12:00', 'temperature_f': 35.0}, om_data=None, owm_data={'timestamp': '2026-01-19T12:00', 'snow_depth_m': 0.05}
   - Output: {'timestamp': '2026-01-19T12:00', 'temperature_f': 35.0, 'snow_depth_m': 0.05}

4. **All sources missing**: Return None or empty dict
   - Input: nws_data=None, om_data=None, owm_data=None
   - Output: None

5. **Overlapping timestamps aligned**: Merge on timestamp field
   - Both sources must have matching timestamp, or merger uses first available timestamp

6. **Type preservation**: Numeric fields stay numeric (float), strings stay strings
  </behavior>
  <implementation>
Use pandas.DataFrame for merging:
1. Convert each non-None source dict to single-row DataFrame
2. Use pd.concat() with outer join to preserve all columns
3. For duplicate columns (e.g., wind_speed_mph_nws and wind_speed_mph_om):
   - Use .fillna() to coalesce: prefer NWS, then Open-Meteo, then OpenWeatherMap
   - Drop source-suffixed columns after coalescing
4. Convert back to dict and return

Follow RESEARCH.md Pattern 3 example code structure.
  </implementation>
</feature>

    - Keep `After=network-online.target`, `Wants=network-online.target`, `User=cta-eta`,

    ## deploy/cta-compaction.timer

    Use the RESEARCH.md Pattern 7 template exactly:

    ```ini
    [Unit]
    Description=CTA Data Compaction Timer
    Requires=cta-compaction.service

    [Timer]
    OnCalendar=America/Chicago *-*-* 03:00:00
    Persistent=true
    Unit=cta-compaction.service

    [Install]
    WantedBy=timers.target
    ```

    Note: `OnCalendar=America/Chicago *-*-* 03:00:00` uses timezone prefix syntax available since
    systemd >= 233 (standard on Debian). See RESEARCH.md Pitfall 1 for DST implications.
    `Persistent=true` ensures the timer fires on next boot if machine was offline at 3am.

    ## New constant


    ## cmd_compaction(args: argparse.Namespace) -> None

    Reads all JSON sidecar files written by compact.py in `data/compaction/` and displays them.

    Logic:
    1. `compaction_dir = _DEFAULT_COMPACTION_DIR` (or from args if `--dir` flag added — keep simple, no flag needed).
    2. Glob `compaction_dir.glob("compaction-*.json")`, sort by filename descending (most recent first).
    3. Apply `--days` limit: only show sidecars with dates within `args.days` days of today.
    4. If no sidecars found: print "No compaction records found in {compaction_dir}".

    Human-readable output format:
    ```
    Compaction Status (last 7 days)
    ================================

    Date        Daemon              Status    Journals  Rows      Upload   Elapsed
    ─────────────────────────────────────────────────────────────────────────────
    2026-02-17  train_positions     success   96        1,423,800 18.2 MB  12.3s
    2026-02-17  weather             success   48        2,304     0.1 MB   1.2s
    2026-02-16  train_positions     success   95        1,401,000 17.9 MB  11.8s
    ...

    Summary: 2 days, 4 runs, 0 failures
    ```

    Columns:
    - Date: `metrics["date"]`
    - Daemon: `metrics["daemon"]`
    - Status: `metrics["status"]` — format "success" as-is, "partial" as "PARTIAL", "failed" as "FAILED"
    - Journals: `metrics["journals_found"]` (with repaired/skipped note if non-zero: "96 (1 repaired)")
    - Rows: `metrics["rows_written"]` formatted with commas
    - Upload: `metrics["upload_bytes"]` formatted as MB (e.g. "18.2 MB")
    - Elapsed: `metrics["elapsed_seconds"]` formatted as "Xs"

    JSON output (`--json` flag): print list of raw sidecar dicts.

    Exit code: `sys.exit(1)` if any run has status="failed" in the displayed window. `sys.exit(0)` otherwise.

    ## _add_compaction_command(subparsers) -> None

    ```python
    def _add_compaction_command(subparsers):
        parser = subparsers.add_parser("compaction", help="Show compaction job status and metrics")
        parser.add_argument(
            "--days", type=int, default=_DEFAULT_DAYS_WINDOW,
            help=f"Time window in days (default: {_DEFAULT_DAYS_WINDOW})"
        )
        parser.add_argument("--json", action="store_true", help="Output as JSON")
        parser.set_defaults(func=cmd_compaction)
    ```

    ## Wire into main()

    In `main()`, add `_add_compaction_command(subparsers)` after the existing `_add_metrics_command` call.

    Keep all existing commands and behavior unchanged.

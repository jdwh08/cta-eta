<task type="auto">
  <name>Task 2: Implement authentication, rate limiting, and security hardening</name>
  <files>src/cta_eta/monitoring/server.py, config.toml</files>
  <action>
Add security layers to FastAPI server:

**1. Bearer token authentication:**
- Read MONITORING_TOKEN from environment variable (.env)
- Create dependency `verify_token()` that checks Authorization: Bearer {token}
- Apply to all endpoints using `Depends(verify_token)`
- Return 401 Unauthorized if missing/invalid token
- Token should be at least 32 characters (validate on startup, log warning if weak)

**2. Rate limiting with slowapi:**
- Use slowapi Limiter with storage backend (in-memory for now)
- Apply rate limits: 60 requests/minute per IP for all endpoints
- Return 429 Too Many Requests when limit exceeded
- Log rate limit violations at INFO level

**3. CORS configuration:**
- Add CORSMiddleware with configurable allowed origins from config.toml
- Default: allow_origins=["http://localhost:3000"] (for future web UI)
- allow_methods=["GET"], allow_headers=["Authorization"]
- AVOID allow_origins=["*"] (security risk)

**4. Input validation:**
- Use Query() for query params with min/max validators
- daemon name: regex pattern ^[a-zA-Z0-9_-]+$ (prevent path traversal)
- limit: min=1, max=500 for api-calls, max=100 for gaps
- Return 400 Bad Request with clear error message on validation failure

**5. Response sanitization:**
- NEVER include API keys, tokens, or credentials in responses
- Sanitize file paths (only return basename, not full paths)
- Limit error message details (no stack traces in production)

**Add config.toml section:**
```toml
[monitoring]
enabled = true
host = "127.0.0.1"  # localhost only by default
port = 8000
allowed_origins = ["http://localhost:3000"]
rate_limit_per_minute = 60
```

Follow existing patterns: Load config via get_config_section("monitoring"), validate on startup, use structured logging for security events.

AVOID exposing internal errors to clients (log detailed errors, return generic 500 messages). AVOID allowing unauthenticated access even to /status (all endpoints require token).
  </action>

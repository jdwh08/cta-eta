  <name>Task 1: Add rolling window metrics to DaemonDiagnostics</name>
  <files>src/cta_eta/data_collection/orchestration/diagnostics.py</files>
  <action>
Extend DaemonDiagnostics class to calculate and track:
- Success rate per span (API/operation): success_count / total_count over time windows
- Error rate per span: error_count / total_count
- Rolling windows: last 1 hour, last 24 hours
- Latency percentiles (p50, p95, p99) per span

Add new method `calculate_metrics()` that returns dict with:
- per_span_metrics: {span_name: {success_rate, error_rate, total_calls, p50_ms, p95_ms, p99_ms}}
- time_window_metrics: {last_hour: {...}, last_24h: {...}}
- overall_health: aggregate health score (all spans combined)

Use time-windowed deques (track timestamp with each event) to calculate rolling windows. Store timestamps with span/error records to enable time-based filtering.

Follow existing patterns: bound memory usage (deque with maxlen), best-effort calculations (never raise on telemetry), use existing _durations_ms infrastructure.

AVOID creating separate data structures for each time window (memory bloat). Instead, filter existing deques by timestamp at calculation time.

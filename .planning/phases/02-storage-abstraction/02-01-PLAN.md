<task type="auto">
  <name>Task 2: Implement Parquet writer with timezone-aware partitioning</name>
  <files>src/cta_eta/storage.py</files>
  <action>
Add ParquetWriter class to src/cta_eta/storage.py that uses PyArrow for Parquet operations.

Constructor takes:
- storage_backend: StorageBackend instance
- partition_hour: int (hour in Chicago time to split days, default 3)
- compression: str (default "snappy")
- timezone: str (default "America/Chicago")

Implement write() method:
- Accepts data as list[dict[str, Any]] (train position records)
- Adds request_timestamp field if not present (current UTC time)
- Converts request_timestamp to America/Chicago timezone using zoneinfo.ZoneInfo
- Determines partition date based on Chicago time:
  - If hour < partition_hour (3 AM), use previous calendar day
  - If hour >= partition_hour, use current calendar day
- Generates Hive-style partition path: date=YYYY-MM-DD/data.parquet
- Converts data to PyArrow Table using pyarrow.Table.from_pylist()
- Writes table to bytes using pyarrow.parquet.write_table() with BytesIO
- Calls storage_backend.put() with partition path and Parquet bytes

Use Python 3.9+ zoneinfo.ZoneInfo for timezone handling (not pytz - zoneinfo is stdlib). Use PyArrow's Table.from_pylist() for dict → Table conversion. Use Hive-style partitioning with date= prefix for compatibility with analytical tools.

DO NOT implement append mode yet - each write() call creates a new file with timestamp suffix (date=YYYY-MM-DD/data_{timestamp}.parquet). DO NOT add deduplication - preserve all data points as specified in PROJECT.md. Keep timestamp in ISO 8601 format with milliseconds.
  </action>

  </verify>

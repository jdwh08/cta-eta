**Tech stack available:**
- AsyncBaseDaemon base class with lifecycle management
- httpx.AsyncClient for HTTP with connection pooling
- aiometer for rate-limited concurrent requests
- stamina retry decorator on API functions
- ParquetWriter with dataset_name support
- DaemonDiagnostics with span timing and event recording
- Structured logging with log_context()

**Established patterns:**
- Daemon initialization: Load config, initialize storage, extract rate limits via get_config_section()
- Daemon run(): Create httpx clients once with timeout/limits, enter while self.running loop
- Error handling: Classify errors with ErrorCategory enum, handle CancelledError, log with extra_fields
- Cycle structure: log_context() wrapper, diagnostics.span() for timing, collection timestamp tracking
- State persistence: Override _get_state() returning dict of metrics
- Rate limiting: aiometer.run_all() with functools.partial for job batching

**Constraining decisions:**
- Phase 5: aiometer rate limiting pattern established for API calls
- Phase 5: dataset_name parameter for ParquetWriter (use "train_positions")
- Phase 2: Hive-style daily partitions at 3 AM Chicago timezone

**CTA API specifics:**
- Rate limit: 50,000 requests/day (~0.58 req/sec sustained, ~34 req/min)
- Target polling: Every 15 seconds = 4 calls/min = 5,760 calls/day (well under limit)
- API endpoint: Single call returns all 8 lines at once (efficient batch fetch)
- Response format: Nested route/train structure, already normalized by normalize_train_positions()

    - Create httpx.AsyncClient with timeout (connect=10s, read=30s, write=10s, pool=10s) and limits (max_connections=10, max_keepalive_connections=5)
    - Use "async with httpx.AsyncClient(...) as client:" context manager
    - Enter "while self.running:" loop
    - Call await self._collect_train_positions_cycle(client) wrapped in try/except
    - Handle asyncio.CancelledError → raise (propagate cancellation)
    - Handle Exception → classify with classify_error(), log appropriately, continue loop
    - Apply backoff on ErrorCategory.RATE_LIMIT (2x poll interval)
    - Exit on ErrorCategory.CONFIGURATION (set self.running = False, raise)
    - Sleep via "await self.sleep(self.train_poll_interval)" (interruptible sleep from AsyncBaseDaemon)

    **async _collect_train_positions_cycle(client) private method:**
    - Generate cycle_id via self.diagnostics.new_cycle_id() if enabled
    - Wrap entire method in log_context(daemon_class=self.__class__.__name__, cycle_id=cycle_id, diag_run_id=self.diagnostics.run_id)
    - Wrap collection logic in "async with self.diagnostics.span('train_positions_cycle', cycle_id=cycle_id):"
    - Record poll_timestamp = time.time() BEFORE API call (preserve exact timing)
    - Call "async with self.diagnostics.span('cta.get_train_positions', cycle_id=cycle_id):"
    - Call raw_response = await get_train_positions(client) inside span
    - Call records = normalize_train_positions(raw_response, datetime.fromtimestamp(poll_timestamp, tz=timezone.utc))
    - Log info: f"Fetched {len(records)} train positions from CTA API" with extra_fields
    - Call self.storage.append_batch(records, dataset_name="train_positions")
    - Update state: self.last_poll_timestamp = poll_timestamp, self.total_records_collected += len(records), self.current_poll_count += 1
    - Log summary with extra_fields: records_stored, cycle_duration_ms, total_records_collected, current_poll_count
    - Handle exceptions: log with self.logger.exception() including cycle_id in extra_fields, re-raise

    **_get_state() method override:**
    - Return dict with: last_poll_timestamp (float), total_records_collected (int), current_poll_count (int), train_poll_interval_seconds (int)

    **Important patterns to follow:**
    - Use async with for httpx client lifecycle (no manual client.aclose())
    - Use self.sleep() not asyncio.sleep() (enables fast shutdown)
    - Always propagate asyncio.CancelledError
    - Use logger.exception() in except blocks (captures stack trace automatically)
    - Use extra_fields dict for structured logging
    - Record diagnostic events via self.diagnostics.record_event() for key operations
    - datetime objects must be timezone-aware (use timezone.utc)
    - Never use time.sleep() (blocking) - only asyncio.sleep() or self.sleep()

    **What to avoid and WHY:**
    - Don't use synchronous httpx.Client → WeatherDaemon uses AsyncClient for non-blocking I/O
    - Don't swallow asyncio.CancelledError → breaks graceful shutdown
    - Don't use bare except: → hides CancelledError and makes debugging harder
    - Don't block event loop with time.sleep() → Use self.sleep() which is interruptible
    - Don't forget timezone on datetime → Parquet schema expects timezone-aware timestamps

<task type="auto">
  <name>Task 2: Create alert runner script</name>
  <files>scripts/run_alerts.py, config.toml, .env.template, pyproject.toml</files>
  <action>
    Create `scripts/run_alerts.py` — executable alert runner that:

    1. Loads config from config.toml [alerting] section (or returns early with message if section missing/disabled).
    2. Builds AlertConfig from config + .env SMTP credentials.
    3. Runs `cta-monitor metrics --json` via `subprocess.run(["cta-monitor", "metrics", "--json"], capture_output=True, text=True)`.
       - If subprocess fails (non-zero exit or no output), logs warning and exits 0 (don't crash cron jobs on missing daemon).
    4. Parses JSON output. Passes metrics_data to `should_send_alert()`.
    5. If should_send_alert returns True:
       - Builds subject: "CTA Daemon Alert: {N} violations"
       - Builds body: format_alert_message(violations) + "\n\nFull metrics:\n" + json.dumps(metrics_data, indent=2)
       - Calls send_email_alert(). On success, calls save_alert_timestamp().
       - Prints "Alert sent: {N} violations" to stdout.
    6. If False: prints "No alert needed" and exits 0.

    Config additions to config.toml:
    ```toml
    [alerting]
    enabled = false  # Set to true to enable email alerts
    cooldown_hours = 4  # Minimum hours between repeat alerts
    last_alert_state = ".daemon_state/last_alert.json"  # Cooldown state file
    smtp_host = "smtp.gmail.com"
    smtp_port = 587
    smtp_from = "alerts@example.com"
    smtp_to = ["oncall@example.com"]  # List of recipients
    ```

    .env.template additions:
    ```
    # SMTP credentials for alerting (if alerting.enabled = true in config.toml)
    SMTP_USERNAME=your_smtp_username_here
    SMTP_PASSWORD=your_smtp_password_here
    ```

    pyproject.toml: Add scripts entry `cta-alerts = "cta_eta.monitoring.run_alerts:main"` — BUT ONLY if the runner is refactored into a proper module. If kept as scripts/run_alerts.py, instead add a shebang line `#!/usr/bin/env python3` and make it importable. Check project convention first: Phase 8 added cta-monitor as a pyproject.toml script entry pointing to a module. Follow same pattern: create `src/cta_eta/monitoring/run_alerts.py` and add entry point `cta-alerts = "cta_eta.monitoring.run_alerts:main"`.

    Wrap entire logic in `main()` function for entry point.

    Config loading: read from config.toml using existing `load_config()` or direct `tomllib.load()` — check how config.py exposes this. Fall back gracefully if [alerting] section absent (print info message, exit 0).

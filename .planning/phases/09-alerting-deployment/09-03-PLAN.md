<task type="auto">
  <name>Task 2: Create health check script</name>
  <files>src/cta_eta/monitoring/health_check.py, pyproject.toml</files>
  <action>
    Create `src/cta_eta/monitoring/health_check.py` with `main()` entry point.

    Logic:
    1. Scan `.daemon_state/*.heartbeat.json` to discover all daemons with heartbeat files.
    2. For each heartbeat file, read `{"timestamp": unix_float, "daemon": name, "pid": int}`.
    3. Check age: `now - timestamp`. Default stale threshold: 600 seconds (10 minutes = 2x diagnostics interval).
    4. Accept `--threshold SECONDS` CLI argument (argparse) to override default.
    5. Accept `--json` flag for machine-readable output (same pattern as cli.py).

    Exit codes (same convention as cta-monitor status):
    - 0: All discovered daemons have fresh heartbeats (or no heartbeat files found but that's OK — just print info)
    - 1: One or more daemons have stale heartbeats (stale = age > threshold)
    - 2: Heartbeat directory missing (.daemon_state/ doesn't exist)

    Human output format:
    ```
    Daemon Health (threshold: 600s)
    ================================
    TrainPositionDaemon   HEALTHY   (42s ago, pid=12345)
    WeatherDaemon         STALE     (750s ago, pid=12346)
    ```

    JSON output (`--json`):
    ```json
    {
      "status": "degraded",
      "threshold_seconds": 600,
      "daemons": [
        {"name": "TrainPositionDaemon", "status": "healthy", "age_seconds": 42, "pid": 12345},
        {"name": "WeatherDaemon", "status": "stale", "age_seconds": 750, "pid": 12346}
      ]
    }
    ```

    Handle missing/corrupt heartbeat files gracefully (skip with warning).

    Add entry point to pyproject.toml: `cta-health = "cta_eta.monitoring.health_check:main"`

    Follow cli.py patterns: `from __future__ import annotations`, ruff noqa comments for T201/PLR2004, type annotations throughout.
  </action>

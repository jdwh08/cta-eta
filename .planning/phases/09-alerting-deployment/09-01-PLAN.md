<feature>
  <name>Alert threshold checking and cooldown logic</name>
  <files>src/cta_eta/monitoring/alerting.py, tests/monitoring/test_alerting.py</files>
  <behavior>
    `should_send_alert(metrics_data: dict, last_alert_path: Path, cooldown_hours: int) -> bool`

    Cases:
    - metrics_data["should_alert"] is True AND last_alert_path does not exist → True (first alert ever)
    - metrics_data["should_alert"] is True AND last alert was > cooldown_hours ago → True
    - metrics_data["should_alert"] is True AND last alert was < cooldown_hours ago → False (in cooldown)
    - metrics_data["should_alert"] is False → False (regardless of cooldown)
    - metrics_data missing "should_alert" key → False (defensive)
    - last_alert_path exists but contains invalid JSON → treat as no previous alert → True if should_alert

    `format_alert_message(violations: list[dict]) -> str`

    Cases:
    - Single violation: returns readable string with violation type and value
    - Multiple violations: returns multi-line string with each violation on its own line
    - Empty violations list: returns "No specific violations reported"
    - Violation with "metric", "threshold", "actual" keys: includes all three in message
    - Violation missing keys: handles gracefully without KeyError

    Alert state persistence:
    - `save_alert_timestamp(last_alert_path: Path) -> None` — writes {"last_alert": unix_timestamp} JSON
    - Uses best-effort I/O (OSError suppressed, logged at debug level)
    - `load_last_alert_time(last_alert_path: Path) -> float | None` — returns None if missing/invalid
  </behavior>
  <implementation>
    Create `src/cta_eta/monitoring/alerting.py` with three public functions.

    `should_send_alert`: Read last_alert_path via `load_last_alert_time()`. If None or (now - last_alert_time) > cooldown_hours * 3600, check metrics_data.get("should_alert", False). Return True only if both conditions met.

    `format_alert_message`: Iterate violations list. For each, format as "- {metric}: actual={actual} exceeds threshold={threshold}" using .get() with defaults. Join with newlines.

    `save_alert_timestamp` / `load_last_alert_time`: JSON read/write to last_alert_path. Wrap I/O in try/except OSError and try/except (json.JSONDecodeError, KeyError) with fallback to None.

    Use `from __future__ import annotations` and type annotations throughout (Python 3.13 style).
    No module-level imports of external packages — pure stdlib (json, time, pathlib).
  </implementation>
</feature>

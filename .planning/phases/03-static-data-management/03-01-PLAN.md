  <action>
Create `src/cta_eta/cache.py` with a generic `CachedData` class:
- Type parameters: `CachedData[T]` using Python 3.13+ generic syntax
- Constructor: `__init__(cache_file: Path, ttl: int, fetch_fn: Callable[[], T])`
- Public method: `get() -> T` - returns cached data, refreshing if expired
- Private methods:
  - `_is_expired() -> bool` - checks if `time.time() - cached_at > ttl`
  - `_load_from_file() -> None` - reads JSON cache file, populates `_memory_cache`
  - `_save_to_file() -> None` - writes `_memory_cache` to JSON with metadata
  - `_refresh() -> None` - calls `fetch_fn()`, updates cache, saves to file
- Attributes: `_cache_file: Path`, `_ttl: int`, `_fetch_fn: Callable[[], T]`, `_memory_cache: dict[str, Any] | None`
- Error handling: Missing cache file on load is normal (returns None, triggers refresh). JSON decode errors log warning and trigger refresh.
- JSON format: `{"data": T, "cached_at": float, "ttl": int}`
- Thread safety: NOT thread-safe initially (single-threaded daemon assumption, can add locks later if needed)

Follow project conventions:
- Snake_case naming
- Type hints for all methods and attributes
- Docstrings for public methods
- Use pathlib.Path (not strings) for file paths
- Modern Python 3.13+ generics (no typing.Generic)

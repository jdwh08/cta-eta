<objective>
Implement lazy discovery weather grid caching that learns API-specific grid identifiers from actual API responses, rather than pre-computing grid points.

Purpose: Let NWS and Open-Meteo APIs naturally determine their own grid systems, cache the API-specific grid identifiers (e.g., "LOT/85,67" for NWS, rounded coords for Open-Meteo), and enable instant lookups during polling without redundant API calls.
Output: API-specific grid mapping caches that populate lazily as stations are encountered, with TTL-based refresh for all mappings.
</objective>

Create `src/cta_eta/weather_grid_cache.py` implementing lazy discovery grid caching:

**Core classes:**

1. `class WeatherGridCache:`
   - Generic interface for API-specific grid caching
   - Methods: `get_grid_identifier(station_id: str, lat: float, lon: float) -> str`
   - Returns cached grid identifier if exists, otherwise discovers from API and caches
   - Uses CachedData[dict[str, str]] for persistence (maps station_id → grid_identifier)

2. `class NWSGridCache(WeatherGridCache):`
   - Lazy discovery for NWS API
   - On cache miss for station: call `get_nws_forecast_url(lat, lon)` to discover gridpoint
   - Extract gridpoint from forecast URL (format: `/gridpoints/LOT/85,67/forecast/hourly`)
   - Parse to get grid identifier (e.g., "LOT/85,67")
   - Cache mapping: station_id → "LOT/85,67"
   - Return grid identifier for future fast lookups

3. `class OpenMeteoGridCache(WeatherGridCache):`
   - Lazy discovery for Open-Meteo API
   - On cache miss for station: make minimal API request to Open-Meteo with station coords
   - Extract actual coordinates used by API from response (they round/snap internally)
   - Cache mapping: station_id → "lat,lon" (API's actual grid point)
   - Return grid identifier for future fast lookups

**Public functions:**

```python
def get_nws_grid_cache(config: dict) -> NWSGridCache:
    """Factory function for NWS grid cache with config-driven setup."""
    cache_file = Path(config["cache"]["directory"]) / "nws_grid_mapping.json"
    ttl = int(config["cache"]["weather_mapping_ttl"])
    return NWSGridCache(cache_file, ttl)

def get_open_meteo_grid_cache(config: dict) -> OpenMeteoGridCache:
    """Factory function for Open-Meteo grid cache with config-driven setup."""
    cache_file = Path(config["cache"]["directory"]) / "open_meteo_grid_mapping.json"
    ttl = int(config["cache"]["weather_mapping_ttl"])
    return OpenMeteoGridCache(cache_file, ttl)
```

**Key behaviors:**

- **Lazy population**: Empty cache on first run, discovers grid IDs as stations are encountered
- **Batch refresh on TTL expiry**: When TTL expires, all mappings are stale; next access triggers re-discovery for that station
- **Persistence**: Uses CachedData[dict[str, str]] for file-backed storage
- **Error handling**: If API discovery fails (stamina retries exhausted), serve stale cache if available, otherwise raise exception

**Type safety:**
- Full type hints with modern Python 3.13+ syntax
- Generic base class for reusability
- No `Any` types except where truly necessary

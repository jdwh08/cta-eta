<objective>
Implement JournalWriter — an Arrow IPC stream writer that appends record batches to local journal files with time-based rotation and hive-style directory paths.

Purpose: Replace per-poll Parquet writes with an append-friendly IPC journal. Each poll appends to the current journal file; after 15 minutes (configurable) the journal rotates to a new file. This eliminates the ~5,760 files/day small-file problem and prepares for Phase 11 compaction.

Output: Tested `JournalWriter` class + `create_journal_writer(config)` factory + `config.toml` journal rotation setting.
</objective>

  <files>src/cta_eta/data_collection/storage_cache/journal_writer.py, tests/data_collection/storage_cache/test_journal_writer.py</files>
  <behavior>
    Core behavior (testable contracts):

    1. **append_batch creates file at correct hive path:**
       Input: records=[{"train_id": "123", "lat": 41.0}], dataset_name="train_positions", data_path=tmp_path
       Expected: file created at `{tmp_path}/train_positions/year=YYYY/month=MM/day=DD/journal_HHMMSS.ipc`

    2. **append_batch writes readable IPC data:**
       Input: records=[{"a": 1}, {"a": 2}], then read back via pa.ipc.open_stream()
       Expected: table with 2 rows and column "a" matching input

    3. **same journal file used within rotation window:**
       Input: two append_batch calls within rotation interval (no manual rotate)
       Expected: both write to the same .ipc file (file path unchanged between calls)

    4. **new journal file created after rotation interval:**
       Input: JournalWriter(rotation_interval_seconds=1), append_batch, sleep(2s), append_batch
       Expected: second call writes to a different .ipc file

    5. **close() flushes and finishes the IPC stream:**
       Input: append_batch then close()
       Expected: file is readable via pa.ipc.open_stream() after close

    6. **manual rotate() closes current file and next append opens new file:**
       Input: append_batch, rotate(), append_batch
       Expected: two separate .ipc files exist

    Edge cases:
    - Empty records list → ValueError("Cannot write empty batch")
    - append_batch after close() → reopens/creates new journal
  </behavior>
  <implementation>
    Class: JournalWriter

    __init__(self, data_path: str | Path, rotation_interval_seconds: int = 900, timezone: str = "America/Chicago")
    - Store data_path as Path, rotation_interval_seconds, ZoneInfo(timezone)
    - Initialize _writer: pa.ipc.RecordBatchWriter | None = None
    - Initialize _current_file: Path | None = None
    - Initialize _journal_start_time: datetime | None = None
    - Initialize _schema: pa.Schema | None = None

    append_batch(self, records: list[dict[str, Any]], dataset_name: str = "default") -> None
    1. Validate records non-empty
    2. Convert records to pa.RecordBatch via pa.RecordBatch.from_pylist(records)
       - On first call: capture schema from batch, store as self._schema
    3. Check if rotation needed: _writer is not None AND elapsed > rotation_interval_seconds
       → call rotate()
    4. If _writer is None: call _open_new_journal(dataset_name)
    5. self._writer.write_batch(batch)

    _open_new_journal(self, dataset_name: str) -> None
    - Compute now = datetime.now(self._timezone)
    - Build path: data_path / dataset_name / f"year={now.year}" / f"month={now.month:02d}" / f"day={now.day:02d}" / f"journal_{now.strftime('%H%M%S')}.ipc"
    - Create parent dirs
    - Open sink = pa.OSFile(str(path), 'wb')
    - self._writer = pa.ipc.new_stream(sink, self._schema)
    - self._current_file = path
    - self._journal_start_time = now
    - Store sink as self._sink for closing

    rotate(self) -> None
    - If _writer is None: return (no-op)
    - self._writer.close(); self._sink.close()
    - Reset _writer = None, _current_file = None, _journal_start_time = None

    close(self) -> None
    - Call rotate() to close cleanly

    Factory: create_journal_writer(config: dict[str, dict[str, Any]]) -> JournalWriter
    - Reads config["storage"]["data_path"] (default "data")
    - Reads config["storage"]["journal_rotation_minutes"] (default 15) → * 60 for seconds
    - Returns JournalWriter(data_path=data_path, rotation_interval_seconds=rotation_seconds)

    config.toml addition (under [storage]):
    # Arrow IPC journal rotation interval in minutes (default 15)
    # After this interval, the current journal file is closed and a new one is opened
    journal_rotation_minutes = 15
  </implementation>
</feature>
